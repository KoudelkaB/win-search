# File Search Manager ヘルプ

`F1` でアプリ内のヘルプを開き、`F12` で NTFS インデックスを更新します。

## フィルター

- `report` は名前に含む、`:report` は先頭、`report:` は末尾、`:report:` は完全一致です。
- `.pdf:|.docx:` はこれらの拡張子で終わる名前を検索し、空白で区切った語は **AND** です。
- `src\` は親フォルダー、`src\\` は完全パスを検索します。
- `"C:\Work"` は直下、`"C:\Work\\"` は再帰的に検索します。

空白を含むパスは引用符で囲みます。`Ctrl+Left`/`Ctrl+Right` は履歴、`Down` は候補を表示します。

## 実用例

- **ディスク容量を空ける** — **フィルター**を空にして、インデックス済みのすべての項目を表示します。**サイズ**の見出しを（必要ならもう一度）クリックし、最大の項目が上に来るようにして、上から順に確認します。不要な項目は `Shift+Delete` で削除できます（ごみ箱を経由せず完全に削除されます）。
- **アプリケーションの書き込み先を監視する** — **フィルター**を空にし、**変更日時**の見出しをクリックして、最新の項目が上に来るようにします。監視したいアプリケーションを起動または使用すると、書き込み中のファイルが変更のたびに上へ移動し、**フォルダー**列で場所を確認できます。ファイルシステムの一部だけを監視する場合は、先に **フィルター**へパスを入力します。
- **ソースファイル内のテキストを検索する** — 例えば `C:\Projects\\ .cs:` でプロジェクト内の `.cs` ファイルに絞り込みます。検索する文字列を **検索**に入力して `Enter` を押します。

## 検索とキー

文字列を入力して `Enter` を押します。`UTF-8` と `UTF-16` はテキスト、`HEX` は `48 65 6C 6C 6F` のようなバイトを検索します。`Delete` は削除、`F2` は名前変更、`O` は開く、`U`/`Z` は展開/圧縮です。`›` はサブメニュー、`Backspace` は戻る、`Esc` は中止です。
