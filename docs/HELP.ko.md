# File Search Manager 도움말

`F1`은 앱 안에서 이 도움말을 열고 `F12`는 NTFS 색인을 새로 고칩니다.

## 필터

- `report`는 이름에 포함, `:report`는 시작, `report:`는 끝, `:report:`는 정확히 일치합니다.
- `.pdf:|.docx:`는 해당 확장자로 끝나는 이름을 찾으며 공백으로 구분한 항목은 **AND**입니다.
- `src\`는 부모 폴더, `src\\`는 전체 경로를 검색합니다.
- 이름 앵커는 경로의 각 구성 요소에 적용되므로 `:src:\\`는 경로의 어느 위치에 있든 이름이 정확히 `src`인 폴더가 포함된 항목을 찾습니다.
- `"C:\Work"`는 바로 아래를, `"C:\Work\\"`는 재귀적으로 검색합니다.

공백이 있는 경로는 따옴표로 묶습니다. `Ctrl+Left`/`Ctrl+Right`는 기록, `Down`은 제안을 표시합니다.

## 실용적인 예

- **디스크 공간 확보** — **필터**를 비워 색인된 모든 항목을 표시합니다. **크기** 열 머리글을 (필요하면 다시) 클릭해 가장 큰 항목이 위에 오도록 한 다음 위에서부터 목록을 검토하고, 필요 없는 항목은 `Shift+Delete`로 삭제합니다(휴지통을 거치지 않고 영구 삭제).
- **애플리케이션의 기록 위치 실시간 확인** — **필터**를 비우고 **변경됨** 열 머리글을 클릭해 가장 최근에 변경된 항목이 위에 오도록 합니다. 관찰할 애플리케이션을 시작하거나 사용하면, 애플리케이션이 기록하는 파일이 변경될 때마다 위로 이동하고 **폴더** 열에서 위치를 확인할 수 있습니다. 파일 시스템의 일부만 관찰하려면 먼저 **필터**에 경로를 입력합니다.
- **소스 파일에서 텍스트 검색** — 예를 들어 `C:\Projects\\ .cs:`를 사용해 프로젝트의 `.cs` 파일로 결과를 제한합니다. 찾을 텍스트를 **검색**에 입력하고 `Enter`를 누릅니다.

## 검색 및 키

텍스트를 입력하고 `Enter`를 누릅니다. `UTF-8`/`UTF-16`은 텍스트, `HEX`는 `48 65 6C 6C 6F` 같은 바이트를 찾습니다. `Delete` 삭제, `F2` 이름 바꾸기, `O` 열기, `U`/`Z` 압축 풀기/만들기입니다. `›`는 하위 메뉴, `Backspace`는 뒤로, `Esc`는 취소입니다.

## 삭제 및 복원

`Delete`는 먼저 선택한 각 폴더 전체를 휴지통으로 이동합니다. 잠겼거나 접근할 수 없는 항목 때문에 실패하면 가능한 가장 큰 하위 폴더부터 처리하고, 필요한 경우 개별 파일까지 계속합니다. 삭제할 수 없는 항목과 이를 담는 데 필요한 상위 폴더는 원래 위치에 남습니다.

휴지통 항목은 특성과 원래 위치를 유지합니다. 수동으로 복사하지 말고 휴지통의 **복원**을 사용하세요. 일부만 처리된 경우 원래 트리가 여러 항목으로 표시될 수 있습니다. `Shift+Delete`는 같은 방식으로 영구 삭제하며 필요할 때 `읽기 전용` 특성을 제거합니다.
